# Comprehensive Audit Report: Methodological Evaluation of Ayurvedic Text Pipeline

## 1. Introduction and Scope
This report provides a rigorous academic audit of the neuro-symbolic NLP pipeline designed for processing archaic Sinhala Ayurvedic manuscripts. The project involves a "Hybrid CRF" for Sentence Boundary Detection (SBD) in Phase 1, an XLM-RoBERTa deep learning comparison in Phase 2, and a Knowledge Graph safety guardrail for clinical toxicity validation. 

While the architectural design (OCR Correction -> SBD -> Safety Guardrail) is theoretically sound and addresses a highly novel, impactful problem in clinical historical NLP, the **empirical execution and evaluation methodology currently contain critical flaws that invalidate the findings**.

To elevate this project to a PhD-level publishable standard (e.g., for venues like ACL, EMNLP, or IEEE ICDAR), the fundamental issues regarding dataset generation and baseline evaluations must be addressed.

---

## 2. Critical Methodological Flaws (The "Why" it's currently failing)

The current pipeline suffers from circular logic and stochastic evaluation, making the reported scores (e.g., 96.6% accuracy) statistically meaningless.

### 2.1 Fatal Flaw in Ground Truth Generation (`gold_test.tsv`)
In `scripts/generate_gold.py`, the "Gold Standard" test set is generated **entirely randomly**. It assigns a `STOP` tag to the end of a line with an 88% probability and to any intermediate word with a 3% probability (`random.random() < 0.03`).
*   **The Issue:** You are evaluating a trained machine learning model against random noise rather than linguistic ground truth. All derived metrics (Accuracy, Precision, Recall, F1), confusion matrices, and statistical significance tests reported in your notebooks are mathematically invalid. 

### 2.2 Circular Training Dynamics (`train_labeled.tsv`)
The training dataset is synthesized via `weak_supervision_generator.py` using a single, deterministic rule: if a word ends in specific Sinhala suffixes (e.g., 'යි', 'යුතු'), tag it as `STOP`.
*   **The Issue:** Your Phase 1 CRF model is then trained on this data utilizing the `is_common_ending` feature, which checks for these exact same suffixes. The model is not learning "Ayurvedic Morphology" from context; it is simply memorizing a trivial script you wrote. You could achieve the exact same result with an `if/else` statement without any machine learning.

### 2.3 Redundant Ablation Study
In `evaluation/evaluate.py`, your ablation study removes the `is_common_ending` feature, but the results (`crf_no_ending` vs `crf_full`) are identical (p-value = 1.0). 
*   **The Issue:** The standard CRF features (`word[-2:]`, `word[-3:]`) already perfectly capture the target suffixes. The claim in your thesis that the "Hybrid" rule injection is the critical driver of your model's success is empirically falsified by your own ablation findings. The "Hybrid" part is redundant.

### 2.4 Biased Model Comparison (Phase 1 vs. Phase 2)
Your conclusion that "CRF > RoBERTa" for this low-resource task is flawed due to the data pipeline.
*   **The Issue:** RoBERTa underperforms because it is a sub-word (BPE) model being forced to learn a deterministic character-suffix rule generated by your weak supervision script. The CRF has an explicit feature tailored to that exact rule. Furthermore, both models are being evaluated on the random noise generated by `generate_gold.py`. This is not a fair or scientifically valid comparison of model architectures.

### 2.5 Inadequate Guardrail Evaluation
The safety guardrail evaluation currently relies on 8 hard-coded test strings within a Python script.
*   **The Issue:** A PhD-level defense requires empirical proof of safety. Eight manually crafted strings are insufficient to calculate true sensitivity, specificity, and False Negative rates in life-or-death clinical applications.

## 3. Actionable Recommendations (The "How to Fix It")

To make this research publishable and defensible at the PhD level, implement the following steps:

### Step 1: Create an Authentic Human-Annotated Gold Standard
**Stop using `generate_gold.py`.** 
*   You must commission an Ayurvedic domain expert (or perform the annotation yourself if qualified) to manually annotate a minimum of 500-1000 authentic sentences from digitized palm-leaf manuscripts to serve as a true SBD test set. This is the absolute minimum requirement for publishable NLP research.

### Step 2: Define Fair and Meaningful Baselines
The deterministic rule-based script you used to generate the training data must serve as your **Primary Baseline**. 
*   Apply the `weak_supervision_generator.py` rules directly to your new human-annotated Gold Standard. 
*   If your CRF or RoBERTa model cannot statistically significantly outperform this simple `if/else` script on the human test set, then the Machine Learning approach is unnecessary. The ML models *must* prove they learn contextual nuance beyond simple suffix matching.

### Step 3: Upgrade the Weak Supervision Pipeline (Optional but Recommended)
If you need to generate 70,000 sentences for RoBERTa and cannot hand-label them, do not use a single deterministic rule. 
*   Use a formal weak supervision framework like **Snorkel**. Combine multiple noisy heuristics (e.g., suffix rules + word length + surrounding context + bigram probabilities) to generate probabilistic labels. This forces the ML models to learn generalized representations rather than memorizing a single rule.

### Step 4: Fix the "Hybrid" Claim
Given your ablation results, you must redefine your feature engineering claims.
*   Acknowledge that standard character n-grams (`word[-2:]`) effectively capture morphology in highly agglutinative languages like Sinhala. 
*   If you want to maintain the "Hybrid" claim, you must engineer linguistic features that standard n-grams *cannot* capture easily (e.g., dictionary lookups for known clinical verb roots, explicit Part-of-Speech tags, or distance to the nearest noun).

### Step 5: Fairly Re-evaluate XLM-RoBERTa
Once you have an authentic, human-annotated test set and a less deterministic training set:
*   Fine-tune RoBERTa again. The comparison between the statistical model (CRF) and the deep learning model (Transformer) is a highly publishable narrative (especially regarding data scarcity), but it is only valid if evaluated on real human ground truth.

### Step 6: Expand the Safety Evaluation Benchmark
Develop a robust benchmark dataset comprising at least **100+ clinical Ayurvedic recipes**.
*   This dataset must include complex edge cases: missing purification steps, distant purification steps (cross-sentence), negated purification (e.g., "do not boil"), and safe vs. toxic herbs. 
*   Calculate the actual Confusion Matrix (TP, TN, FP, FN) on this dataset to empirically prove the efficacy of your Sliding Window guardrail.

## Conclusion
The conceptual framework of this pipeline-combining probabilistic linguistic boundaries with deterministic medical knowledge graphs-is excellent and highly relevant. However, the entire evaluation pipeline is currently compromised by synthetic, rule-based training data and randomly generated test data. By replacing the synthetic data with human-annotated ground truth and establishing fair baselines, you will transform this from a flawed experiment into a rigorous, publishable PhD thesis.
